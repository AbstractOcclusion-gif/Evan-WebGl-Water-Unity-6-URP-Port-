# WebGpuWater Package Audit — 2026-07-02

Scope: `Packages/com.abstractocclusion.webgpuwater` — 15 Runtime C#, 4 Editor C#, 16 shader/compute/hlsl files (~7,400 lines). Read-only audit: safety/robustness, coding standards, shader/WebGPU/mobile correctness. Top findings spot-verified against source.

**No Critical findings.** Nothing crashes or corrupts on the happy path. The High items surface on enable/disable lifecycle churn and WebGPU-build-only sampling paths.

## C. Coding standards compliance

Scorecard: comments (WHY) **good** · guard clauses **good** · composition **good** · magic numbers **mixed** · hardcoded strings **mixed** · naming **mixed** · SRP **mixed** (one offender) · fail-fast **mixed** · dead code **mixed** · public API surface **poor**.

Top violations:

1. **WaterVolume.cs is a god class** — 1,649 lines, ~10 responsibilities (sim driving, caustics, registry/scheduler, input routing, sim window, bed baking, readback/sampling, uniform publishing, camera framing). The code admits it (line 636 TODO). Extract `WaterInputRouter`, `WaterSimScheduler`, `WaterSurfaceSampler`, `WaterBedBaker`. `OnEnable` alone does ~10 jobs in ~80 lines.
2. **~120 public mutable fields across 8 components** — WaterProbe/WaterRippleEmitter prove the right pattern (`[SerializeField] private` + accessors); apply package-wide. `WaterSimulation`, `WaterObstacle`, `WaterWaveBank` could be `internal`.
3. **Magic-number clusters**: WaterSplashEmitter (~25 literals), WaterBuildKit (~40, incl. camera constants duplicating WaterVolume's consts as raw literals — drift risk), WaterQuality default-tier literals duplicated, epsilon soup in WaterVolume (`1e-3/1e-5/1e-6` unnamed).
4. **Stale hardcoded string**: WaterFoamParticles.cs:133–135 error message references a menu path that no longer exists (`Tools > WebGL Water > …` vs actual `AbstractOcclusion/WebGpuWater/Water Wizard`). Scene-object names duplicated inline across WaterBuildKit and WaterSceneBuilder.
5. **Dead code**: 4 unused WaterBuildKit methods (`SaveMaterial`, `CreateProceduralTerrain`, `CreateStaticObstacle`, `CreateFloatingProp`), vestigial `MODE_*` int consts (should be an enum; value 1 gap is a removed mode), WaterSphere.shader.
6. **Naming**: `_ctrl` (stale "controller"), `_a/_b`, `_cb`, `sfWater`, plus a log message referencing the old "Water Controller" class name (WaterSceneBuilder.cs:182).

## D. What's done well (verified)

Capability gating with clean self-disable; analytic-waterline fallback when AsyncGPUReadback is unsupported; single in-flight readback with reused CPU buffer; cached PropertyToIDs everywhere and near-zero per-frame allocations; symmetric static registry add/remove; `Sanitize` divergence firewall + mean-correction clamp; explicit-LOD sampling in divergent loops; write-only `Dst` vs sampled `Src`; WaterWaveBank's constant documentation (JONSWAP factors with derivations) is best-in-class; WaterSimulation.cs is the model file for the whole package.

## E. Suggested fix order

1. **Lifecycle pass** (F1–F4): material/RT destroy discipline — one small PR, removes both Highs.
2. **WebGPU sampling pass** (W1, W3, W4): extend the manual-bilinear fix to foam mask + caustics; point-filter HeightMip.
3. **W2 + W5**: RHalf footprint RT; decide compat-mode story for foam particles (likely fixes the open particle build bug).
4. **Mobile perf tier** (P1–P4): quality-tier defines for refine steps, wave count, god-ray res/shadows; half precision migration.
5. **Standards batch**: delete dead code, const-ify splash/buildkit literals, fix stale strings, MODE enum.
6. **Structural** (larger): split WaterVolume; encapsulation pass; CG→URP HLSL migration (unlocks SRP batcher).

Items 5–6 fold in existing backlog #10/#11/#21/#14.
